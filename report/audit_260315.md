## 2. 코드 품질 이슈 (3건)

### 2-A. `ACADEMIC_THRESHOLDS` 불필요한 빈 줄

`evidence.py:634-668` — `ACADEMIC_THRESHOLDS` 정의에 **매 필드 사이에 빈 줄**이 삽입되어 가독성이 떨어집니다.

```diff
 ACADEMIC_THRESHOLDS = ClassificationThresholds(
-
     dc_raw_min=0.70,
-
     dc_ident_min=0.60,
-
     sso_raw_max=0.15,
+    dc_raw_min=0.70,
+    dc_ident_min=0.60,
+    sso_raw_max=0.15,
     ...
 )
```

### 2-B. `_extract_identifiers()`에 `extension` 파라미터 미전달

`evidence.py:77-95` — `_extract_identifiers(source)` 호출 시 `extension` 인자를 받지 않습니다. 내부의 `_is_noise_identifier(t)` 호출에서 **항상 기본값 `.java`** 가 사용되어, Python/JS 등 비-Java 파일에서도 `String`, `Object` 등 Java 타입 stopword가 필터링됩니다.

- **영향**: Java 외 언어에서 `String`이라는 식별자가 존재하면 불필요하게 제거됨
- **해결**: `_extract_identifiers(source, extension=".java")` → 호출처에서 extension 전달

### 2-C. Floating docstring 경고

`evidence.py:630-632`, `evidence.py:670` — `INDUSTRIAL_THRESHOLDS`, `ACADEMIC_THRESHOLDS` 인스턴스 **다음 줄**에 docstring이 위치합니다. Python에서 모듈/클래스/함수의 docstring만 유효하고, **변수 뒤 docstring은 런타임에 무시**됩니다 (IDE에서만 힌트).
